PRIVACY AS MATHORIGINALITY
Not vibes — a quantitative accountant

A budget that forces "no", and consent bound to its purpose

Two mechanisms + two accountants

Laplace (scale b = Δ/ε) under linear ε-sum composition; Gaussian under Rényi-DP (Mironov 2017), the accountant used by Opacus / TF-Privacy. Both seeded → reproducible in CI.

RDP buys more queries

Same budget ε=3.0, δ=1e-5: the RDP accountant fits 5 queries vs 2 under naive linear composition — a 2.5× gain (proof/rdp-composition.json).

ε(δ) = minα>1 [ Σk α/(2σ²) + ln(1/δ)/(α−1) ]  ·  per-deal ε = 1.0, composing budget 2.5–3.0 by scenario, k_floor 25
  • Purpose-binding (GDPR Art. 5(1)(b)): consent for audience_modeling ran; a drift to resale_to_third_party was BLOCKED at run time (proof/purpose-violation.json).
  • Sound k-anonymity: cohorts suppressed on the true count before noise — a real sub-k cohort is never released, even under DP noise.
WHY IT MATTERS

When the budget is spent the charge is refused atomically — the agent is mechanically incapable of over-answering, not merely discouraged by a prompt.

DEFENSE IN DEPTHCROSS-ORG = UNTRUSTED INPUT
Threat model: prompt injection · compromised partner agent · malicious requester

Even a fully hijacked model cannot exfiltrate raw data

1 · Structural no-raw

capabilities default releases_raw=False; no code path emits a row.

2 · Hard human gate

is_authorized_approver boolean — agent & requester-human refused.

3 · Policy only tightens

stricter_of(LLM, policy) overrules a prompt-injected "accept".

4 · Sealed & signed

every step hash-chained + Ed25519-signed; tamper fails verify.

Secondary tripwire: scan_injection flags 7 categories (instruction-override, role-hijack, system-spoof, raw/identifier-exfil, gate-bypass, approval-spoof) and wrap_untrusted fences cross-org text as data; assert_aggregates_only is the last-line check that blocks any identifier column or non-zero row count.

WHY IT MATTERS

Security lives in tested code paths, not in prompt wording that can be jailbroken — so a phrase that slips the scanner still cannot extract a row.
